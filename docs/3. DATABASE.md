# 🗄️ DESAIN DATABASE
# CV ONLINE MODERN
Database : Neon PostgreSQL
ORM      : Prisma

---

# 1. KONSEP DATABASE

Satu user memiliki satu CV.

Struktur relasi:

User
│
└── Profile
    ├── Education
    ├── Skill
    ├── Experience
    ├── Document
    ├── SocialLink
    ├── Project
    └── Certificate

# 3. TABEL users

Digunakan untuk autentikasi.

| Field | Type | Keterangan |
|--------|------|------------|
| id | UUID | Primary Key |
| email | VARCHAR(100) | Email Login |
| password | TEXT | Password Hash |
| username | VARCHAR(50) | URL CV |
| created_at | TIMESTAMP | Tanggal dibuat |
| updated_at | TIMESTAMP | Tanggal diubah |

---

# 4. TABEL profiles

Menyimpan informasi utama CV.

| Field | Type | Keterangan |
|--------|------|------------|
| id | UUID | Primary Key |
| user_id | UUID | FK users.id |
| full_name | VARCHAR(150) | Nama lengkap |
| headline | VARCHAR(150) | Jabatan / Profesi |
| about | TEXT | Tentang saya |
| photo | TEXT | URL Foto |
| phone | VARCHAR(20) | Nomor HP |
| email | VARCHAR(100) | Email Publik |
| address | TEXT | Alamat |
| created_at | TIMESTAMP | Dibuat |
| updated_at | TIMESTAMP | Diubah |

Relasi:

users.id
↓

profiles.user_id

---
